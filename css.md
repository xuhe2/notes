# 优先级



## 样式表的优先级

* 只有冲突的部分会需要使用优先级的顺序关系.
* 优先级是看`<style>`的部分.

> 样式表分成
>
> 1. 行内的样式表,直接在写标签中的
> 2. 内部的优先级,写在当前的`HTML`文件的`<style>`中的
> 3. 外部的优先级,写在单独的`CSS`文件中的

* `行内`>`内部`=`外部`,但是同级别的部分依赖在文件中的顺序关系.

# 选择器



## 通配选择器

> 使用`*`,可以代表全部的选择器



## 元素选择器

> 使用元素的名字,对文件中全部的这个元素进行处理



## 类选择器

* 不同的标签也可以是同一个分类
* 在写代码前需要一个`.`表示是一个类
* 可以把多个类都写在一个`class=`中



## `ID`选择器

* 前面使用`#`开始,代表是`ID`.



## 交集选择器

* 只有同时满足条件的时候才使用

>  `.元素名`+`.类名`,代表你的代码是在同时满足以上条件的时候才运行.



## 并集选择器

* 条件满足其中一个的时候,都可以使用.
* `或`的符号是`,`.

> 格式是`.类名`+`,`+`.类名`+`,`+`.类名`.



## 后代选择器

> 中间使用`空格`切分.(可以写多个)

* 注意查找的是后代,不是子部分.



## 子代选择器

> 使用`>`.



## 兄弟选择器



### 使用`+`(相邻兄弟的选择器)

* 注意,不仅仅是同级别的兄弟的标签,并且这个兄弟一定是`往下一个`找到的(找不到的话,那么就不使用).





### 使用`~`(通用兄弟选择器)

* 后面全部的兄弟级别的符合条件的都变化.



## 属性选择器

* 对于写明使用某种属性的标签

```css
    <style>
        [title] 
        {
            color: red;
        }
    </style>
```



* 如果需要有使用这个属性,并且属性值是某个确定的数值的标签

```css
    <style>
        [title="XXX"] 
        {
            color: red;
        }
    </style>
```



* 对于有这个属性,并且这个属性值需要使用某个东西作为靠头的标签

```css
    <style>
        [title^="XXX"] 
        {
            color: red;
        }
    </style>
```



* 对于使用什么部分结尾的属性值的标签

```css
    <style>
        [title$="XXX"] 
        {
            color: red;
        }
    </style>
```



* 对于只需要包含一个部分的属性值的标签

```css
    <style>
        [title*="XXX"] 
        {
            color: red;
        }
    </style>
```



## 伪类选择器

> 使用`:`作为分割,后面的部分是状态.
>
> 很像类,但其实是特殊状态的一种表示.



### 动态伪类

> 悬浮在某个控件的时候,使用`:hover`
>
> 按下的时候,`:active`
>
> 访问过的,`:visited`
>
> 没有访问过的,`:link` 
>
> 鼠标点击,获取焦点(这个空间必须有可以被选中的状态)`:focus`



### 结构伪类

* 寻找子代,并且使用结构化的修改
* 注意,在这里,查找的是`<p>`的部分.
* 可以在括号里面写公式,比如`2n`,`-n+5`(注意,n从0开始,形式必须是`an+b`)
* 可以在括号里面写`even`,`odd`等参数.

```css
        div>p:nth-child(1) {
            color: red;
        }

        div>p:nth-child(2) {
            color: green;
        }

        div>p:nth-child(3) {
            color: blue;
        }
```

* 当你指定了类型之后,你需要保证这个类型确实是在下一个位置,如果类型错误,不会被执行.



* 查找特定的类型,所有的同类型的兄弟

``````css
div>p:first-of-type{}
``````

* 也有`nth-of-type()`



* 选中作为唯一的子类的元素

> 使用`only-child`的



* 选中内容是空的元素

# 盒子模型

> 有`外边框`,`边框`,`内边框`,`内容` 四个部分组成.

1. `margin`
2. `border`
3. `padding`
4. `content`

* `margin`只能影响盒子的位置,不能影响盒子大小



## `content`

> 使用`width`设置的是固定的
>
> 可以使用`min-width`设置最小的部分,当小到下限的时候,就不会变小了
>
> 也有`max-width`



## `padding`

> 可以单独设置某一侧的方面

> 可以使用`padding-left`,`padding-top`....修改

> 可以直接使用`padding`写多个值

* 注意,行内元素的上下内边距可能出问题



## `border`

> 默认的边框的大小是`3px`,可能会因为缩放问题不同

> 自己设置的大小不受缩放的影响



## `margin`

* 注意事项

1. 修改的时候,只会影响后面和自己,不会影响先前的控件部分

2. 对于行内元素来说,修改上下的`margin`大小是没有效果的



### `margin`塌陷问题

> 对于处于边上的子元素的`margin`的大小修改会被父元素拿走

> 解决办法:
>
> 1. 使父元素具有一定的`border`样式
> 2. 使`padding`具有一定的大小(`0px`不行)
> 3. 设置`overflow`为`hidden`,控制溢出部分的设置



### `margin`合并问题

> 两个控件的`margin`部分不会被分割开,会合并在一起

* 别动这个问题,不用解决



# 处理内容溢出问题

> 当容器里面的部分溢出的时候,可以在容器层面使用`overflow`属性
>
> 1. hidden 
> 2. visible
> 3. scroll,使用`滚动条`.



# 隐藏元素

1. `display: none`不占位
2. `visibility: hidden`:占位



# 继承样式

* 继承的样式的优先级很低.



# 元素之间的空白问题

> 因为行内元素是在同一行的,在代码中的换行占一个小空格

> 父元素设置`font-size=0px`,因为空格也是字符,需要在子元素中设置字体大小.



# 行内块元素的空白问题

> 设置`vertical-align`,原来的是按照`baseline`的模式.



# 浮动

> 浮动之后,所有的显示方式都要按照`浮动`的方式来显示.
